Application Description:

A windows form in c# framework 4.0 has been created. It contains 2 rich text boxes one for view/edit purposes
and a second one to display "error" messages.

Main functionality for this application is to use "PurchaseOrderSchema.xsd" schema located at the project level
and validate different xml files, or input via rich text box against this schema.
Only valid xml data will be allowed to be saved.
In order to be able to save, you must first have to open a valid XML file.
Different exception handling errors were implemented and displayed via messagebox.

Unit tests were implemented for each method as requested based on NUint.
For GUI testing i could not implement NUint forms, but I created an automation script that tests all possible variants of GUI.

Source code XML comments are being saved under ".../bin/Debug(or Release)/XML_Configuration_Editor.XML" file.

"PurchaseOrderSchema.zip" contains original xml files, included just in case the ones included at project level got corrupted/broken.

IMPORTANT NOTES:


If needed add NUnit-2.6.2 dll and NUnitFormsV2.0.0.5alpha4 dll as references in project. (NUnitFormsV2.0.0.5alpha4 namespace was removed)
